# AI Platform 使用文档

## 1. 项目简介

AI Platform 是一个前后端分离的多能力 AI 工作台，当前包含以下模块：

- SQL 优化与历史记录
- 图片生成
- 视频生成
- 语音生成
- 3D 镜头控制
- 商品营销文案生成
- 广告语生成
- 营销历史记录
- 模板中心、任务中心、素材中心

项目结构：

- 后端：Spring Boot 3 + MyBatis-Plus + MySQL + Redis
- 前端：Vue 3 + Vue Router + Vite

---

## 2. 环境要求

建议本地环境：

- JDK 17
- Maven 3.9+
- Node.js 18+
- MySQL 8.x
- Redis 6.x 或更高

---

## 3. 目录说明

主要目录如下：

- `src/main/java`：后端业务代码
- `src/main/resources`：后端配置与初始化 SQL
- `frontend/src`：前端页面与组件
- `frontend/public`：前端静态资源与兼容页面
- `data/uploads`：本地上传文件默认保存目录
- `deploy`：部署相关示例配置
- `docs`：项目文档

## 4. 本地配置

### 4.1 后端配置

先复制本地配置模板：

```powershell
Copy-Item application-local.example.yml application-local.yml
```

然后按实际环境修改 [application-local.example.yml](C:/Users/luoyn/Desktop/aiset/application-local.example.yml) 中对应项：

- `spring.datasource.username`：MySQL 用户名
- `spring.datasource.password`：MySQL 密码
- `aliyun.api-key`：阿里云 DashScope Key
- `aliyun.proxy.enabled`：是否通过 OpenAI 兼容代理调用模型
- `aliyun.proxy.model`：代理默认模型，当前示例为 `gpt-5.4`
- `aliyun.proxy.base-url`：代理地址
- `aliyun.proxy.api-key`：代理 API Key
- `app.storage.upload-dir`：上传文件目录

默认主配置见 [application.yml](C:/Users/luoyn/Desktop/aiset/src/main/resources/application.yml)。

### 4.2 前端配置

如需自定义前端接口地址，可参考 [frontend/.env.example](C:/Users/luoyn/Desktop/aiset/frontend/.env.example)：

```env
VITE_API_BASE_URL=http://localhost:8080
VITE_ASSET_BASE_URL=http://localhost:8080
```

开发环境通常不需要单独配置，因为 Vite 已代理：

- `/api` -> `http://localhost:8080`
- `/uploads` -> `http://localhost:8080`

## 5. 数据库初始化

项目启动时会执行 [schema.sql](C:/Users/luoyn/Desktop/aiset/src/main/resources/schema.sql)。

默认数据库名：

- `ai_sql_platform`

主要表：

- `app_user`
- `sql_optimize_record`
- `image_generate_record`
- `voice_generate_record`
- `camera_generate_record`
- `video_generate_record`
- `marketing_generate_record`

如果你是首次运行，确保：

1. MySQL 服务已启动
2. 配置中的账号具备建库建表权限
3. Redis 服务可连接

## 6. 启动项目

### 6.1 启动后端

在项目根目录执行：

```powershell
mvn spring-boot:run
```

后端默认地址：

- `http://localhost:8080`

常用路径：

- API：`http://localhost:8080/api`
- 上传资源：`http://localhost:8080/uploads`

### 6.2 启动前端

在 `frontend` 目录执行：

```powershell
npm install
npm run dev
```

前端默认地址：

- `http://localhost:5173`

### 7.1 构建前端

```powershell

npm run build
```

- `frontend/dist`

## 8. 登录与账号使用

### 8.1 注册

打开：

- `/#/register`

输入用户名和密码后即可注册，注册成功会自动建立会话。

### 8.2 登录

打开：

- `/#/login`

输入用户名和密码后登录。

登录页已支持“记住密码”：

- 勾选后会把账号和密码保存在当前浏览器的 `localStorage`
- 下次打开登录页会自动回填
- 如取消勾选，浏览器中的记住信息会被清除

说明：

- 当前登录态由后端 Session + Redis 维护
- 前端会在访问受保护页面时自动校验登录状态

## 9. 页面与功能说明

前端采用 Hash 路由，主页面入口见 [frontend/src/router/index.js](C:/Users/luoyn/Desktop/aiset/frontend/src/router/index.js)。

### 9.1 SQL 优化

页面：

- `/#/`

用途：

- 输入 SQL 与数据库类型
- 调用后端进行 SQL 分析和优化建议生成
- 查看优化评分、诊断结果和历史记录

相关接口：

- `POST /api/sql/optimize`
- `GET /api/sql/history`
- `GET /api/sql/history/{id}`

### 9.2 图片生成

页面：

- `/#/image`

用途：

- 通过文本提示词生成图片
- 支持使用本地上传图片作为参考图
- 可查看图片生成历史

相关接口：

- `POST /api/upload/image`
- `POST /api/image/generate`
- `GET /api/image/history`

### 9.3 视频生成

页面：

- `/#/video`

用途：

- 提交视频生成任务
- 查询异步任务状态
- 查看视频生成历史

相关接口：

- `POST /api/video/generate`
- `GET /api/video/task/{taskId}`
- `GET /api/video/history`

### 9.4 语音生成

页面：

- `/#/voice`

用途：

- 输入文本生成语音
- 查看生成历史

相关接口：

- `POST /api/voice/generate`
- `GET /api/voice/history`

### 9.5 3D 镜头控制

页面：

- `/#/camera`

用途：

- 上传原始图片
- 调整方位角、俯仰角、距离
- 生成不同视角图像

相关接口：

- `POST /api/camera/generate`
- `GET /api/camera/history`

### 9.6 营销文案生成

页面：

- `/#/marketing`

用途：

- 上传商品图
- 填写商品名称、类目、卖点、目标人群
- 选择平台、内容风格、输出长度、CTA
- 生成小红书、抖音、B 站营销文案
- 自动生成附加结果，如广告语、海报标题、生图提示词、短视频分镜

页面附加能力：

- 可加载示例数据
- 可按平台分 Tab 查看结果
- 支持单块复制与整页复制
- 支持在当前页面直接生成营销海报图片
- 支持跳转到图片工作台继续深度调整

默认模型说明：

- 营销文案与广告语生成默认走 `gpt-5.4`
- 海报图片生成仍使用图片模型 `qwen-image-2.0`

相关接口：

- `POST /api/marketing/copy`
- `POST /api/upload/image`
- `POST /api/image/generate`

### 9.7 广告语生成

页面：

- `/#/slogan`

用途：

- 输入产品名称、类目、卖点、目标人群、品牌调性
- 生成 slogan、广告语、电商主图文案、短视频标题等内容

相关接口：

- `POST /api/marketing/slogan`

### 9.8 营销历史

页面：

- `/#/marketing-history`

用途：

- 查看营销文案与广告语历史记录
- 支持筛选、查看、复制、删除
- 可从历史中回填并继续编辑

相关接口：

- `GET /api/marketing/history`
- `DELETE /api/marketing/history/{id}`

### 9.9 模板中心

页面：

- `/#/templates`

用途：

- 保存常用生成配置
- 在营销、图片等工作流之间复用模板

### 9.10 素材中心

页面：

- `/#/assets`

用途：

- 集中查看生成后的图片、音频、视频等素材

### 9.11 任务中心

页面：

- `/#/tasks`

用途：

- 查看各类生成任务进度与结果

## 10. 营销中心操作流程

如果你主要使用营销能力，推荐按下面流程操作：

1. 进入 `/#/marketing`
2. 上传商品图
3. 填写产品名称、类目、核心卖点、目标人群
4. 选择平台、风格、输出长度和 CTA
5. 点击“生成文案”
6. 在右侧查看小红书、抖音、B 站和附加结果
7. 如需海报，直接在下方“海报生成”区域点击“使用 AI 提示词”
8. 点击“生成海报图片”
9. 如需更细致调整，点击“打开图片工作台”
10. 在 `/#/marketing-history` 查看和复用历史结果

## 12. 常见问题

### 12.1 前端能打开，但接口都失败

排查顺序：

1. 确认后端已启动
2. 确认 `http://localhost:8080/api/auth/me` 可以访问
3. 确认 Vite 代理或 `VITE_API_BASE_URL` 配置正确
4. 检查浏览器控制台和后端日志

### 12.2 登录后仍然跳回登录页

可能原因：

- Redis 未启动，Session 无法持久化
- 浏览器禁用了 Cookie
- 前后端地址不一致导致会话未带上

### 12.3 图片上传成功但无法显示

检查：

- `app.storage.upload-dir` 是否可写
- `/uploads/**` 是否被后端正确暴露
- `VITE_ASSET_BASE_URL` 是否配置正确

### 12.4 营销文案生成失败

检查：

- 代理模型配置是否可用
- `aliyun.proxy.base-url` 和 `aliyun.proxy.api-key` 是否正确
- 如果不用代理，确认 `aliyun.api-key` 是否有效

### 12.5 端口冲突

默认端口：

- 前端 `5173`
- 后端 `8080`
- Redis `6379`
- MySQL `3306`

如有冲突，请修改对应服务配置。
